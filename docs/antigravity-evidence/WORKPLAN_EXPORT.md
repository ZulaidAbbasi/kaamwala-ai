## Workplan Design Decisions

**Why 20 phases?**
Each phase delivers a complete, testable increment. This allows:
- Clean TypeScript build after every phase
- Incremental verification (logged in VERIFICATION_LOG.md)
- No phase depends on a future phase (forward-only dependency)

**Why backend-first?**
- API endpoints are tested independently before mobile screens
- Enables fallback behavior testing without UI dependency
- Secrets stay in backend from day one
