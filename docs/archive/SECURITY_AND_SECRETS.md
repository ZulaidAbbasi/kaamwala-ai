# KaamWala AI — Security & Secrets Management (v3)

**Last Updated:** 2026-05-16

---

## Rule #1

> **If a key can be used to make billable API calls, it is a SECRET and must stay server-side.**

---

## Classification Table

| Secret | Classification | Where It Lives | Why |
|--------|---------------|---------------|-----|
| Gemini API Key | 🔴 SECRET | `firebase functions:config:set gemini.api_key` | Direct billing access |
| Google Maps/Places API Key | 🔴 SECRET | `firebase functions:config:set maps.api_key` | Direct billing access |
| Service Account JSON | 🔴 SECRET | **Never download** — Cloud Functions auto-auth | Full project admin access |
| Firebase Client API Key | 📗 PUBLIC | `.env` → `EXPO_PUBLIC_FIREBASE_API_KEY` | Client identifier only, no direct API access |
| Firebase Project ID | 📗 PUBLIC | `.env` → `EXPO_PUBLIC_FIREBASE_PROJECT_ID` | Project identifier |
| Firebase Auth Domain | 📗 PUBLIC | `.env` → `EXPO_PUBLIC_FIREBASE_AUTH_DOMAIN` | Auth URL |
| Cloud Functions URL | 📗 PUBLIC | `.env` → `EXPO_PUBLIC_API_BASE_URL` | Public HTTPS endpoint |

---

## Mobile App Security Checklist

```
✅ Uses Firebase public config only (from .env)
✅ Calls our backend, never Google APIs directly
✅ Gets auth token from Firebase Anonymous Auth
✅ Sends auth token in Authorization header
✅ Reads Firestore with security rules (user-scoped)
✅ Cannot write to Firestore directly (Admin SDK only in backend)
❌ Does NOT contain Gemini API key
❌ Does NOT contain Maps API key
❌ Does NOT contain service account credentials
❌ Does NOT make direct Google Places/Maps/Gemini calls
```

---

## Backend Security Checklist

```
✅ API keys from functions.config() at runtime
✅ Validates auth token on every request
✅ Writes to Firestore via Admin SDK (auto-credentials)
✅ Logs agent traces with no secret data
✅ Returns error objects, not stack traces
❌ Does NOT log API keys
❌ Does NOT return API keys in responses
❌ Does NOT store API keys in source code
```

## Environment Variable Setup

### Mobile App (.env)

```env
# PUBLIC — Firebase client identifiers
EXPO_PUBLIC_FIREBASE_API_KEY=AIzaSy...
EXPO_PUBLIC_FIREBASE_AUTH_DOMAIN=kaamwala-ai.firebaseapp.com
EXPO_PUBLIC_FIREBASE_PROJECT_ID=kaamwala-ai
EXPO_PUBLIC_FIREBASE_STORAGE_BUCKET=kaamwala-ai.firebasestorage.app
EXPO_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=000000000
EXPO_PUBLIC_FIREBASE_APP_ID=1:000:web:abc
EXPO_PUBLIC_FIREBASE_MEASUREMENT_ID=G-XXXXXXX

# PUBLIC — Our backend URL
EXPO_PUBLIC_API_BASE_URL=https://us-central1-kaamwala-ai.cloudfunctions.net/api
```

### Backend (Firebase Functions Config)

```bash
# SECRET — Set once, never commit
firebase functions:config:set gemini.api_key="YOUR_GEMINI_KEY"
firebase functions:config:set maps.api_key="YOUR_MAPS_KEY"

# Verify
firebase functions:config:get
```

### Local Development (Emulator)

```json
// functions/.runtimeconfig.json (auto-generated by emulator)

  "maps": { "api_key": "local-dev-key" }
```

## Pre-Submission Security Scan

Run these before every commit:

```powershell
# Check for hardcoded API keys
Select-String -Recurse -Include *.ts,*.tsx,*.json,*.md -Pattern 'AIzaSy[A-Za-z0-9_-]{30,}' | Where { $_.Path -notmatch 'node_modules' }

# Check for .env files
Get-ChildItem -Recurse -Filter ".env" | Where { $_.FullName -notmatch 'node_modules' }

# Check for service account files
Get-ChildItem -Recurse -Filter "*service-account*" | Where { $_.FullName -notmatch 'node_modules' }
```

All three must return **zero results**.

## What Happens If Keys Leak

| Scenario | Impact | Response |
|----------|--------|----------|
| Gemini key in client code | Anyone can make Gemini calls billed to you | Revoke key immediately in AI Studio |
| Maps key in client code | Unlimited Places/Geocoding calls | Revoke and rotate in GCP Console |
| Service account JSON committed | Full admin access to all Firebase data | Revoke in IAM, rotate, delete commits |
| .env committed to GitHub | All local secrets exposed | Rotate all keys, force-push cleaned history |
